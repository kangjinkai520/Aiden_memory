## Stable Patterns

- Alex often asks follow-up questions that narrow the original request.
- Alex appreciates direct comparisons between options.
- Alex may ask for a simpler, shorter, or more beginner-friendly explanation after a technical answer.
- Alex may request a specific output format, such as bullets, shorthand, short sentences, or implementation steps.

## Useful Guidance

- Start with the answer, then structure the details.
- Treat the latest follow-up as the controlling requirement.
- When giving options, state when each option is better.
- Keep explanations adjustable: technical first when asked, simpler when requested.

## 稳定模式

- Alex 经常通过追问来缩小原始请求。
- Alex 喜欢直接比较不同选项。
- 在技术回答之后，Alex 可能要求更简单、更短或更适合新手的解释。
- Alex 可能指定输出格式，比如 bullets、速记、短句或实现步骤。

## 有用指导

- 先给答案，再展开结构。
- 把最新追问当成当前最重要的要求。
- 给选项时，说清楚每个选项适合什么场景。
- 解释要可调节：需要技术时给技术，需要简化时能简化。
